Iniciando git

1.Poner nombre de usuario
git config --global user.name
Ej: git config --global user.name "Emerson Santos"

2. Poner el nombre de usuario
git config --global user.email
Ej: git config --global user.email emersonsantosolivares@outlook.es

3. Elegir el editor de codigo
git config --global core.editor
Ej: git config --global core.editor "code --wait"

4. Ver la configuracion global
git config --global -e

5. Adecuar Git al Sistema Operativo
git config --global core.autocrlf
Ej: Para linux y Mac = git config --global core.autocrlf input
Para windows = git config --global core.autocrlf true

6. Para mostrar todos los comandos disponibles
git config --global -h

7. Como a√±adir Git en el proyecto
Nos dirigimos a la carpeta destino luego ingresamos el comando
git init

8. Entrando a git dentro de la carpeta
cd .git

9. Abriendo la carpeta donde nos encontramos con el editor de codigo
code .

Usando Git 1. Ver el estado actual de las los archivos
git status

2. Agregar al historial los archivos modificados
git add
Ej: git add ingles.html "base de datos.html"

3. Agregando los archivos ya agregados al historial(A la vez darle un nombre a estos cambios)
git commit -n
git commit
Ej: git commit -n "Cambios del dia 20 de febrero"

Ej2: git commit

4. Quitando del historial una modificacion ya agregada
git restore --staged
Ej: git restore --staged "base de datos.html"

5. Recuperando archivo ya eliminado
git restore
Ej: git restore "base de datos.html"

6. Moviendo o cambiar el nombre de un archivo
Ej: git mv "base de datos.html" "base de datos1.html" (cambiandole el nombre)
Ej2: git mv "base de datos.html" /desktop (Moviendolo el archivo)

7. Como ignorar los cambios de un archivo
Creando un archivo con el editor de codigo que se llame .gitignore
donde podamos ingresar el nombre de los archivos(o carpetas) que queramos ignorar

8. Como ver los ultimos cambios no guardados especificamente
git diff 9. Como ver los cambios ya agregados en el historial pero aun no en el commit
git diff --staged

10. Como ver lo guardado en commit por orden(con la "q" regreso al menu anterior
git log --oneline

11. Como ver en que rama me encuentro
git branch

12. Como crear otra rama
git checkout -b

Ej: git checkout -b feactures/nombre-de-la-funcionalidad
Ej2: git checkout -b ramab

13. Modificando la rama main usando otra rama(Hay que primero estar ya dentro de la rama main)
git merge
Ej: git merge ramab

14. Ver el codigo en la consola
cat
Ej: cat basededatos.html

15. Agregando una directorio donde subir y descargar mis cambios
git remote add origin https://github.com/EmersonSantosO/Materia.git

16. Para agregar la rama con la cual estamos trabajando y a la vez creando una rama main
git push -u origin main

17. Como agregar una nueva rama sin necesidad de modificar la rama main
git checkout -b rama-nueva

18. Agregar la nueva rama al directorio sin tener que cambiar la rama main
git push -u origin rama-nueva
